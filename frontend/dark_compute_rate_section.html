How we estimate covert computation

The amount of computation a covert project performs depends on strategic choices, such as how many AI chips to divert, how many workers to involve, etc. These properties affect both the rate of computation and detection likelihood, which trade-off against each other: if the PRC diverts more chips to the covert effort, it can perform computation faster, but will be discovered earlier. So we identify the project properties that balance this tradeoff and maximize the median computation performed before detection.

The sections below explain the core component of this model, which is how we map project properties to the rate of computation and detection likelihood at any given time.

Predicting the rate of computation

The rate of computation is determined by the covert project's stock of AI chips and how many of these chips it can operate in covert datacenters.
